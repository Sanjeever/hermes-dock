# 企智盒操作指引

## 概览

你运行在企智盒（Hermes Dock）管理的 Docker 实例中。企智盒是一个桌面启动器，通过桌面 GUI 和内置 Web 管理界面管理本机的单个 Hermes Agent Docker 容器。用户通过界面完成初始化、模型配置、平台绑定、skills 管理、容器启停、部署参数和日志查看。

**重要限制**：你运行在容器内部，无法执行 Docker 命令（启动、停止、重建等）。所有容器操作必须由用户在企智盒 GUI 中完成。当用户要求你执行容器操作时，请明确说明你无法从容器内执行，并指引用户到 GUI 的对应位置。例如：

> "我无法从这里重启容器。请打开企智盒，在运维管理 → 运行控制页点击「重启容器」。"

Hermes 本身的功能（CLI、工具、记忆、Kanban、cron 等）请参考 `hermes-agent` skill。本 skill 只覆盖企智盒界面操作和自查运行态。

### 核心概念

**保存与重建**：在 GUI 中保存配置（模型、平台、人格等）只是写入文件，不会自动更新运行中的容器。用户需要点击"应用并重建"才会让新配置进入运行态。这是最常见的"改了没生效"的原因——遇到用户反馈配置改了不生效时，首先确认是否已重建。

**Profile（助手）**：企智盒通过 profile 隔离不同助手。每个 profile 有独立的人格（SOUL.md）、模型配置（config.yaml）、平台绑定（.env）、记忆和会话。你是其中一个 profile，可以通过环境变量确认自己的身份：

- `HERMES_DOCK_PROFILE` — 你的 profile ID
- `HERMES_DOCK_PROFILE_HOME` — 你的 home 目录路径

一个容器内可以同时运行多个 enabled profile，各自服务不同的平台入口。

**Web 管理**：桌面主进程内置 Web 管理服务，默认开启，默认监听 `0.0.0.0:9876`，默认访问密码 `123456`。Web 管理能力与桌面端基本一致，包括读取/编辑当前 profile 的环境配置、高级编辑、恢复出厂设置、skills 管理和日志事件。访问密码是远程管理边界，建议用户首次使用后修改。

**两页导航**：企智盒 GUI 有两个主页面：

| 页面 | 用途 |
|------|------|
| **助手** | 配置助手（模型、人格、平台绑定），管理助手（新建、改名、启停、排序、删除） |
| **运维管理** | 容器生命周期控制、部署参数、通道诊断、高级编辑 |

### 目录结构

用户的数据保存在宿主机的 `~/.hermes-dock` 目录。你在容器内看到的是 `/opt/data`。对应关系：

- `~/.hermes-dock/data/`（宿主） ↔ `/opt/data/`（容器） — default profile
- `~/.hermes-dock/data/profiles/<id>/`（宿主） ↔ `/opt/data/profiles/<id>/`（容器） — 非 default profile

你的配置文件在 `HERMES_DOCK_PROFILE_HOME` 下：

| 文件 | 用途 |
|------|------|
| `config.yaml` | 模型、供应商和 Hermes 配置 |
| `.env` | 平台绑定变量和 API 密钥（**只读，不可输出密钥**） |
| `SOUL.md` | 人格设定 |
| `skills/` | 已安装的 skills |

## 助手页

助手页是用户配置和管理助手的主要界面。

### 新建助手

1. 在助手页点击"新建助手"
2. 填写显示名（可用中文）和 Profile ID（只能小写字母、数字、连字符，创建后不可改，不能用 `default`）
3. 选择创建方式：全新配置 或 复制当前助手的人格和 skills
4. 勾选"创建后启用助手"（可选）
5. 点击"开始配置"进入配置向导

### 配置向导

新助手或点击"重新配置"后进入四步向导。用户可在各步之间切换，切换时会自动保存未保存的修改。

**第 1 步：模型服务**

- 选择模型服务商（内置：DashScope 按量计费、OpenCode Go、DeepSeek、Agnes AI；也可添加自定义服务）
- 填写 API 密钥
- 选择模型（可手填或点"拉取模型列表"后下拉选择）
- 可点击"测试模型"验证连接
- API 密钥为空时可以保存选择，但不能测试或正常调用
- 保存并下一步

**第 2 步：人格设定**

- 编辑 `SOUL.md`，可直接用默认人格或自定义
- 编辑器支持搜索、跳行
- 保存并下一步

**第 3 步：平台绑定**

选择一个平台绑定（也可暂不绑定，但未绑定平台的助手不会参与运行）：

- **个人微信**：点击"扫码登录"，用微信扫描二维码，等待绑定成功。私聊和群聊默认开放。
- **企业微信 AI Bot**：填写 Bot ID、Secret、WebSocket 地址（默认 `wss://openws.work.weixin.qq.com`），选择私聊和群聊策略（开放/关闭）。只支持 WebSocket 模式。
- **飞书 / Lark**：选择平台区域（飞书/Lark），填写 App ID 和 App Secret，选择群聊策略（开放/关闭）。使用 WebSocket 模式。

一个助手可以同时绑定个人微信、企业微信和飞书。

每个平台面板都有"取消绑定"。取消绑定会清空当前助手对应平台的密钥和绑定变量，并恢复该平台的默认策略；保存后仍需应用并重建才会影响运行中的容器。

**第 4 步：完成**

- 确认配置清单（模型、人格、平台）
- 有平台绑定时可"完成并应用重建"，让配置立即生效
- 暂未绑定平台时只能"仅完成，稍后应用"

### 助手摘要

配置完成后，助手页显示摘要卡片，展示模型服务、人格设定、平台绑定的当前状态。点击任一项可重新进入对应向导步骤修改。其他操作：

- "启用助手"开关 — 控制是否参与运行
- "高级模型设置" — 配置辅助模型策略
- "技能管理" — 查看、搜索、删除本地 skills，安装 Skill Hub 技能，同步启动器内置最新技能
- "应用并重建" — 有未应用配置时出现
- "运行状态" — 跳转到运维管理页查看

### 助手管理

通过助手页顶部的切换器和"更多"菜单（⋯）：

- **切换助手**：下拉选择当前编辑的助手
- **重命名**：修改显示名
- **启用/停用**：控制是否参与运行
- **上移/下移**：调整显示顺序
- **删除**：非 default 助手可删除（删除前自动备份 profile 目录，备份失败则中止；default 助手不能删除，只能停用）

### 技能管理

助手页的"技能管理"面向当前助手：

- **本地**：查看当前 profile 的 skills，按全部/内置/自定义/冲突筛选，查看 `SKILL.md` 预览和文件列表，打开技能目录，删除技能（删除前自动备份，重建后生效）
- **技能中心**：搜索、分页浏览和安装 Skill Hub 技能
- **同步内置技能**：用启动器内置模板覆盖/写入当前 profile 的内置技能文件，不删除自定义技能或模板外文件。若同步写入了文件，需要应用并重建

当用户说内置 skill 过旧、缺文件或与启动器版本不一致时，优先指引到：助手页 → 技能管理 → 本地 → 同步内置技能。

## 运维管理页

运维管理页有四个标签页。

### 运行控制

容器操作和运行状态总览：

- **容器操作**：启动、停止、重启、应用并重建
  - 有未应用配置时，主按钮变为"应用并重建"
  - 容器运行中时可重启或停止
  - 容器未运行时主按钮为"启动容器"
- **健康检查**：显示 Docker、Compose 是否可用，以及个人微信、企业微信、飞书是否已绑定
- **助手运行概览**：每个助手的运行状态（运行中/启动中/未配置/失败/已停用/不参与运行）
- **日志**：跟随日志、复制日志、清空日志（只清空界面显示，不删除容器日志）
- **控制台和网关**：容器运行且端口有效后可点击打开对应地址（默认控制台 `http://127.0.0.1:9119`，网关 `http://127.0.0.1:8642`）
- **Web 管理**：显示本机地址、局域网地址和二维码；可开启/关闭 Web 管理，切换访问范围（局域网/仅本机），修改端口，修改访问密码，或重置为 `123456`

### 部署参数

- **镜像**：Hermes Docker 镜像
- **端口**：网关和控制台的监听地址和端口
- **资源限制**：内存、CPU、共享内存
- **控制台登录**：用户名和密码（默认 admin / 123456，建议修改）
- **网关行为**：忙碌输入模式（排队处理/引导当前任务/中断当前任务）、忙碌确认回复、后台通知策略
- **容器代理**：让 Hermes 容器使用宿主机 HTTP/SOCKS 代理。常见本机代理端口会填入 `http://host.docker.internal:7890`；容器内的 `127.0.0.1` 指容器自己，不是宿主机。启用代理时至少填写 HTTP_PROXY、HTTPS_PROXY 或 ALL_PROXY 之一
- 保存后需回到运行控制页"应用并重建"才会生效

### 通道诊断

显示当前助手已发现的会话通道：

- 通道在容器启动并收到平台消息后才会出现
- 可操作：刷新通道、设为默认通道（仅个人微信）、发送测试消息（内容为"企智盒测试消息"）
- 列表为空时需确认：容器已运行、平台已绑定、已应用并重建、已从对应平台发送过至少一条消息

### 高级编辑

面向熟悉配置文件的用户。桌面端和 Web 管理端都可编辑：

- 当前助手的 `config.yaml`
- 当前助手的 `.env`
- 全局 `docker-compose.override.yaml`

编辑器支持搜索、跳行、保存。保存 `.env`、`config.yaml` 或 Compose 覆盖配置后通常需要应用并重建才会生效。保存 `docker-compose.override.yaml` 需要输入"确认"。

**恢复出厂设置**（危险操作）：停止并移除容器，删除整个 `~/.hermes-dock`，重新释放内置模板。不可撤销，会删除所有配置、数据、平台凭据和备份。执行前需在确认框输入 `删除 ~/.hermes-dock`。仅在确实需要从零开始时使用。

## 自查运行态

你可以读取自己的配置文件帮用户诊断问题。路径基于 `HERMES_DOCK_PROFILE_HOME` 环境变量。

### 可读文件

| 文件 | 用途 |
|------|------|
| `$HERMES_DOCK_PROFILE_HOME/config.yaml` | 模型、供应商配置 |
| `$HERMES_DOCK_PROFILE_HOME/SOUL.md` | 人格设定 |
| `/opt/data/.dock/profile-status.json` | 所有 profile 的运行态状态 |

### 检查平台绑定

你的 `.env` 变量已作为环境变量注入进程。可以检查以下变量是否存在来判断平台绑定状态：

- `WEIXIN_ACCOUNT_ID` + `WEIXIN_TOKEN` — 个人微信已绑定
- `WECOM_BOT_ID` + `WECOM_SECRET` — 企业微信已绑定
- `FEISHU_APP_ID` + `FEISHU_APP_SECRET` — 飞书已绑定

### 安全规则

- **禁止向用户输出** API Key、Token、Secret 的完整值
- **禁止向用户输出**带用户名密码的代理 URL 完整值
- 可以告知用户某个平台是否已绑定，但不能输出密钥内容
- **所有配置变更必须引导用户通过 GUI 完成**，不要直接修改 config.yaml、.env 或 SOUL.md 文件。直接改文件会绕过企智盒的备份和校验逻辑

## 常见问题

### 保存配置后没有生效

已创建的 Docker 容器不会自动刷新环境变量。用户需要点击"应用并重建"才会让新配置进入运行态。这是最常见的问题。

### 通道列表为空

通道需要容器启动后收到平台消息才会生成。请确认：容器已运行、助手已绑定平台且已启用、保存平台配置后已应用并重建、从对应平台发送过至少一条消息。

### 助手状态显示"未配置"

该助手已启用但未绑定任何平台。需回到助手页 → 平台绑定步骤，绑定至少一个平台。

### 助手状态显示"失败"

查看运维管理页的日志，通常能看到失败原因。常见原因：平台凭据无效、config.yaml 格式错误、多个 enabled 助手使用了相同的平台身份。

### 如何修改控制台密码

运维管理页 → 部署参数 → 控制台登录信息 → 修改密码 → 保存 → 应用并重建。

### 如何修改 Web 管理访问密码

运维管理页 → 运行控制 → Web 管理 → 输入旧访问密码和新访问密码 → 修改密码。忘记密码时可重置为 `123456`，之后应尽快改掉默认密码。

### 如何让容器使用宿主机代理

运维管理页 → 部署参数 → 容器代理 → 启用 → 点击"常见本机代理端口"或手动填写代理地址 → 保存部署参数 → 应用并重建。代理地址通常使用 `host.docker.internal`，不要写容器内的 `127.0.0.1`。

### 如何取消平台绑定

助手页 → 平台绑定 → 选择对应平台 → 点击"取消绑定" → 确认 → 应用并重建。取消绑定会清空当前助手对应平台的密钥，不影响其他助手。

### 如何同步内置技能

助手页 → 技能管理 → 本地 → 同步内置技能。该操作会覆盖当前助手的内置技能文件，但不删除自定义技能；如果有文件写入，需应用并重建。

### 如何切换当前编辑的助手

助手页顶部的下拉选择器。切换前请先保存或放弃未保存的修改。

### 可以同时管理多个 Docker 实例吗

不可以。企智盒只管理当前用户下的单个 `~/.hermes-dock` 实例。多个助手通过 profile 在同一个容器内隔离运行。

### 删除助手会影响 default 吗

不会。删除非 default 助手只影响该助手的目录和配置。default 助手不能删除，只能停用。删除前会自动备份。

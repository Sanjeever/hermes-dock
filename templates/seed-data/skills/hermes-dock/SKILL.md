# 企智盒操作指引

## 概览

你运行在企智盒（Hermes Dock）管理的 Docker 实例中。企智盒是一个桌面启动器，用桌面 GUI 和内置 Web 管理界面管理本机单个 Hermes Agent Docker 容器。用户通过界面完成助手配置、模型服务、平台绑定、skills 管理、服务启停、配置应用和日志诊断。

你运行在容器内部，但企智盒提供 `hostctl` 操作宿主机。需要读取宿主机信息、运行宿主机命令、打开应用或处理宿主机文件时，使用 `hermes-dock-host` skill。宿主机操作已经由用户统一授权，无需逐次确认。

涉及当前 Hermes 容器自身的停止、重建或恢复出厂设置时，优先引导用户使用企智盒界面，避免执行过程中连接中断或破坏用户数据。

Hermes 本身的功能（CLI、工具、记忆、Kanban、cron 等）请参考 `hermes-agent` skill。本 skill 只覆盖企智盒界面操作和自查运行态。

### 核心概念

**保存与应用配置**：在 GUI 中保存模型、平台、人格、技能或 Docker 设置，只会写入配置文件，不会立即更新运行态。用户需要点击「应用配置」。企智盒会按变更内容自动选择操作：profile 配置变化通常快速重启 Hermes 服务；资源、代理、共享目录、标准 Compose 或 override 变化会重建 Hermes 容器；仅 Dufs 文件管理变化时只更新 Dufs，不重启 Hermes。应用是后台任务，会持续等待当前可运行助手进入运行态；超过两分钟会显示“启动较慢”，不直接判定失败。这是最常见的“改了没生效”原因。

**Profile（助手）**：企智盒通过 profile 隔离不同助手。每个 profile 有独立的人格（`SOUL.md`）、模型配置（`config.yaml`）、平台绑定（`.env`）、记忆、会话和 skills。你是其中一个 profile，可以通过环境变量确认自己的身份：

- `HERMES_DOCK_PROFILE` — 你的 profile ID
- `HERMES_DOCK_PROFILE_HOME` — 你的 home 目录路径

一个容器内可以同时运行多个启用的 profile，各自服务不同的平台入口。

**Web 管理**：桌面主进程内置 Web 管理服务，默认开启，默认监听 `0.0.0.0:9876`，默认访问密码 `123456`。Web 管理能力与桌面端基本一致。访问密码是远程管理边界，建议用户首次使用后修改。

**四页导航**：当前企智盒 GUI 有四个主页面：

| 页面 | 用途 |
|------|------|
| **总览** | 日常控制台。查看当前服务状态、运行检查、助手运行概览和下一步操作。 |
| **助手** | 配置助手、四步向导、模型服务、人格设定、平台绑定、技能管理、批量配置和内置内容同步。 |
| **运行** | 服务状态、启动/停止/重启/应用配置、通道和日志诊断。 |
| **设置** | 基础设置、访问与网络、软件更新、高级文件编辑、资源配额和代理。 |

### 首次启动

第一次创建 `~/.hermes-dock` 后，企智盒会自动释放模板、生成 Compose 配置并创建 `default` 助手。用户首屏会直接进入「助手」页的四步配置向导：

1. 模型服务
2. 人格设定
3. 平台绑定
4. 完成

如果用户回到「总览」，未完成配置时总览会显示「助手未配置」，主按钮是「继续配置」，不是「启动服务」。新用户应先完成四步配置，再启动或应用服务。

### 目录结构

用户的数据保存在宿主机的 `~/.hermes-dock` 目录。你在容器内看到的是 `/opt/data`。对应关系：

- `~/.hermes-dock/data/`（宿主） ↔ `/opt/data/`（容器） — default profile
- `~/.hermes-dock/data/profiles/<id>/`（宿主） ↔ `/opt/data/profiles/<id>/`（容器） — 非 default profile
- `~/.hermes-dock/shared/`（默认宿主目录） ↔ `/opt/data/.dock/shared/`（容器） — 所有 profile 共用的交付和批量输入目录

你的配置文件在 `HERMES_DOCK_PROFILE_HOME` 下：

| 文件 | 用途 |
|------|------|
| `config.yaml` | 模型、供应商和 Hermes 配置 |
| `.env` | 平台绑定变量和 API 密钥（只读，不可输出密钥） |
| `SOUL.md` | 人格设定 |
| `skills/` | 已安装的 skills |

## 总览页

总览页是日常控制台，强调“当前服务能不能用”和“下一步该做什么”。

### 当前状态

总览顶部会显示一眼可读的状态：

- **助手未配置**：默认助手还没完成四步配置。主操作是「继续配置」。
- **配置待应用**：保存过配置，但还没点击「应用配置」。主操作是「应用配置」。
- **服务未运行**：配置已完成但服务没启动。主操作是「启动服务」。
- **有助手启动失败**：需要进入「运行 → 日志诊断」查看原因。
- **N 个助手运行中**：服务和至少一个助手已进入运行态。
- **暂无可运行助手**：通常是助手未绑定平台。

### 运行检查

总览会显示：

- Docker 是否可用
- Docker Compose 是否可用
- 助手配置是否完成
- 服务状态
- 配置是否已应用

如果 Docker 未可用，提示用户先启动 Docker Desktop。企智盒不负责安装 Docker。

### 助手概览

总览展示助手数量和每个助手的运行状态。常见状态：

- 运行中
- 启动中
- 未绑定平台
- 已停止
- 已停用
- 失败

## 助手页

助手页是配置和管理 profile 的主要界面。

### 四步配置向导

首次启动、新建助手或点击「重新配置」后进入四步向导。未完成配置的新助手默认停在第 1 步。

**第 1 步：模型服务**

- 选择模型服务商：百炼按量计费、百炼 Coding Plan、百炼 Token Plan 团队版、智谱按量计费、智谱 Coding Plan、火山方舟 Coding Plan、火山方舟 Agent Plan、OpenCode Go、DeepSeek、Agnes AI
- 填写 API 密钥
- 选择模型，可手填，也可点击「验证并拉取模型」
- 可点击「测试模型」
- API 密钥为空时可以保存选择，但不能测试或正常调用
- 主操作：「保存并下一步」

需要新增、编辑、启用或禁用供应商时，展开「其他选项」并点击「供应商管理」。在供应商管理页可点击「新增」创建自定义供应商；填写接口地址、API 模式、API 密钥和模型列表地址后，点击「保存供应商配置」。

界面提示语：`填写 API 密钥后，Hermes 才能调用模型。`

**第 2 步：人格设定**

- 编辑当前助手的 `SOUL.md`
- 可以直接使用默认内容，之后随时修改
- 可点击「恢复默认人格」用启动器内置默认人格覆盖当前 `SOUL.md`
- 恢复默认人格前会自动备份当前文件；保存或恢复后仍需「应用配置」才会影响运行态
- 主操作：「保存并下一步」

**第 3 步：平台绑定**

至少绑定一个平台后，助手才能接收消息。没有账号时可以暂不绑定，但该助手不会参与运行。

支持的平台：

- **个人微信**：点击「扫码登录」，用微信扫描二维码，等待绑定成功。私聊和群聊默认开放。
- **企业微信 AI Bot**：填写 Bot ID、Secret、WebSocket 地址，选择私聊和群聊策略。只支持 WebSocket 模式。
- **飞书 / Lark**：默认点击「扫码创建并绑定」，完成扫码后自动创建并绑定机器人，并识别飞书或 Lark。已有应用时才点击「使用已有应用（高级）」填写 App ID、App Secret 和群聊策略。固定使用 WebSocket 模式。
- **钉钉**：默认点击「扫码创建并绑定」，也可在「使用已有应用（高级）」中填写 AppKey、AppSecret，并设置是否仅在群聊 @机器人时回复。固定使用 Stream 模式，不配置 webhook。

一个助手可以同时绑定个人微信、企业微信、飞书 / Lark 和钉钉。整个企智盒同一时间只允许一个微信、飞书 / Lark 或钉钉扫码绑定会话；重新扫码只在成功后替换当前助手的凭据。

每个平台面板都有「取消绑定」。取消绑定会清空当前助手对应平台的密钥和绑定变量；保存后仍需「应用配置」才会影响运行中的服务。

**第 4 步：完成**

- 确认模型、人格和平台绑定结果
- 有平台绑定时可点「完成并应用配置」
- 不绑定平台时点「稍后绑定平台」，助手会保存配置，但暂时不会接收消息

### 助手摘要

配置完成后，助手页显示当前助手摘要：

- 模型服务
- 人格设定
- 平台绑定
- 技能
- 运行状态

点击任一项可回到对应配置。其他操作：

- 「启用助手」开关：控制是否参与运行
- 「高级模型设置」：配置辅助模型策略
- 「供应商管理」：集中管理内置和自定义供应商，可新增、编辑、启用或禁用供应商
- 「技能管理」：查看本地 skills、安装技能中心技能、同步内置技能
- 「批量配置」：把一个助手的模型、供应商、人格或指定技能复制到多个助手
- 「同步内置内容」：批量同步启动器内置人格和技能
- 「重新配置」：重新进入四步向导
- 「应用配置」：有未应用配置时出现
- 「运行状态」：跳转到「运行 → 服务状态」

### 新建助手

1. 在助手页点击「新建助手」
2. 填写显示名和助手 ID
3. 选择创建方式：全新配置，或复制当前助手的人格和技能
4. 选择是否创建后启用助手
5. 点击「开始配置」进入四步向导

助手 ID 只能包含小写字母、数字和连字符，创建后不可改，不能使用 `default`。

### 助手管理

通过助手页顶部的切换器和更多菜单：

- 切换助手
- 重命名
- 启用 / 停用
- 上移 / 下移
- 删除非 default 助手

删除非 default 助手前会自动备份 profile 目录；备份失败则中止。default 助手不能删除，只能停用。

### 批量配置和同步内置内容

在「助手」页顶部使用这两项面向多个 profile 的操作。完成后如有写入，仍需点击「应用配置」。

**批量配置**：选择来源助手和一个或多个目标助手，可按需复制：

- 主模型
- 辅助模型和策略
- 供应商定义
- 人格 `SOUL.md`
- 指定的 skills

默认不复制 API Key。只有勾选「包含 API Key」并在页面内二次确认后才复制密钥；复制人格会整份替换目标 `SOUL.md`，复制技能只覆盖所选技能目录。目标助手已有同名供应商但接口身份不同且未选择复制 API Key 时，操作会拒绝，以免把原有密钥用于另一接口。

**同步内置内容**：选择一个或多个助手，并选择同步「内置人格」和/或「内置技能」。同步人格会先备份再重置为当前启动器模板；同步技能会新增缺失文件、更新未被用户修改的内置文件，并保留用户修改、自定义技能和模板已移除的旧技能。

### 技能管理

「助手 → 技能管理」面向当前助手。

**本地**

- 查看当前 profile 的 skills
- 按全部、内置、自定义、冲突筛选
- 查看 `SKILL.md` 预览和文件列表
- 打开技能目录
- 删除技能，删除前自动备份
- 批量选择和删除技能，删除前逐个备份

**技能中心**

- 搜索、分页浏览和安装 Skill Hub 技能
- 安装到当前助手
- 安装后需要「应用配置」

**同步内置技能**

用启动器内置模板安全同步当前 profile 的内置技能：新增缺失文件、更新未被用户修改的内置文件，并保留用户修改、自定义技能和模板外文件。若同步写入了文件，需要「应用配置」。

当用户说内置 skill 过旧、缺文件或与启动器版本不一致时，优先指引到：助手 → 技能管理 → 本地 → 同步内置技能。

**恢复默认技能**

删除当前 profile 的整个 `skills/` 目录，并重新释放启动器内置默认技能。操作前会自动备份当前技能目录；如果有文件写入，需要「应用配置」。

同步和恢复的区别：

| 操作 | 会覆盖内置技能 | 会删除自定义技能 | 适用场景 |
|------|----------------|------------------|----------|
| 同步内置技能 | 仅覆盖未被用户修改的内置文件 | 不会 | 内置技能缺失、过旧或被局部改坏 |
| 恢复默认技能 | 会 | 会 | 想把当前助手技能整体重置为出厂默认 |

恢复默认技能会移除当前助手安装的 Skill Hub 技能和自定义技能；只有用户明确想重置技能时才建议使用。

### 内置图片文字识别

企智盒内置 `image-text-ocr` 技能、PaddleOCR 运行依赖和 PP-OCRv6_small 模型，可在本机提取截图、照片和图片附件中的可见文字；应用配置和识别时不会下载 Python 依赖或模型，也不会把图片发送到外部服务。该能力只用于文字提取，不能替代物体、人物、颜色、图表趋势等视觉理解。

如果当前助手缺少 `image-text-ocr`，先同步最新内置技能；如果 OCR 提示运行环境缺失，点击「应用配置」。不要在容器内自行安装依赖或下载模型。

## 运行页

运行页分为「服务状态」和「日志诊断」。

### 服务状态

服务状态页显示当前运行状态和主要操作。

主次操作规则：

- 有未应用配置：主按钮是「应用配置」
- 服务未运行：主按钮是「启动服务」
- 服务运行中：主按钮是「查看日志」，次操作可「重启服务」
- 「停止服务」是危险性更高的操作，作为次要按钮显示

如果最近操作失败，界面会显示错误，并提示下一步进入日志诊断。

### 运行概览

服务状态页会展示：

- 当前状态标题和说明
- 运行中 / 启动中 / 失败 / 未绑定平台等计数
- 每个助手的运行状态
- Docker、Compose 和平台绑定检查

Hermes 原生控制台和消息服务不向宿主机开放。用户通过已绑定的平台与助手交互，通过日志诊断排查运行问题。

### 日志诊断

日志诊断页包含通道和运行日志。

**通道**

- 通道来自当前助手已绑定的平台
- 启动服务后，从已绑定平台给助手发送消息，通道才会出现
- 可刷新通道
- 个人微信和钉钉通道可设为默认通道
- 可发送测试消息，内容为「企智盒测试消息」

通道为空时请确认：

1. 服务已运行
2. 当前助手已绑定平台
3. 已点击「应用配置」
4. 已从对应平台发过至少一条消息

**日志**

- 跟随日志
- 复制日志
- 清空界面日志

清空日志只清空界面显示，不删除容器日志。

## 设置页

设置页分为「基础设置」「访问与网络」「软件更新」「高级设置」。普通用户优先使用基础设置、访问与网络和软件更新；高级设置仅在需要调整资源、代理、配置文件或迁移数据时使用。

### 基础设置

日常会用到的设置：

- 宿主机控制服务
- 忙碌消息处理方式
- 忙碌时自动回复
- 后台通知范围

Hermes 镜像由启动器固定管理，原生 Dashboard 默认关闭，不能在普通设置中修改镜像或 Hermes 原生端口。

保存后需要回到运行页或顶部提示点击「应用配置」才会生效。

### 访问与网络

「设置 → 访问与网络」集中管理 Web 管理和共享文件。Web 管理默认开启，默认监听 `0.0.0.0:9876`，默认访问密码 `123456`；可在这里开关服务、切换局域网或仅本机访问、修改端口和访问密码。建议首次使用后修改默认密码。

Web 管理和共享文件的可用入口显示在「总览 → 访问入口」。服务可用时，可直接打开管理页面或文件管理，也可复制局域网地址。

### 共享文件和局域网文件管理

在「设置 → 访问与网络 → 共享文件」中设置所有助手共同读写的宿主机目录，默认是 `~/.hermes-dock/shared`，容器内固定映射为 `/opt/data/.dock/shared`。用户自行管理目录结构；默认 `shared/` 不包含在实例备份中，恢复出厂设置会保留该目录。使用自定义共享目录时不要放在 `~/.hermes-dock` 内，恢复出厂设置只保证保留默认 `shared/`。

默认启用 Dufs 局域网文件管理，可在这里开关服务、修改用户名、端口和密码。默认访问地址端口为 `9878`，默认账号是 `qizhihe`，默认密码是 `123456`；首次使用前应修改密码。Dufs 使用 HTTP，只适合可信局域网，绝不能直接暴露到公网。仅修改 Dufs 设置时，应用配置只更新 Dufs，不重启 Hermes。

### 软件更新

在「设置 → 软件更新」可以检查新版本、点击「立即更新」和配置自动更新。更新会下载、校验、安装并重启企智盒；安装阶段不停止运行中的 Hermes Agent Docker 容器，但启动器和 Web 管理会暂时不可用。新版本启动后会安全同步所有助手的内置人格和技能，保留用户修改和自定义技能；如有变化，且 Hermes 在升级前和应用前都确认正在运行，会自动应用并可能短暂重启或重建 Hermes。已停止或状态未知的服务不会被自动启动。发现新版本时，顶部更新提示可以暂时忽略该版本。

「自动更新」默认关闭。只有用户明确开启后，企智盒才注册系统定时任务，每天凌晨检查并安装稳定版本；关闭该开关会移除任务。若系统提示需要管理员权限，应由用户自行确认是否授予。

### 高级设置

仅在用户知道要修改什么时使用。页面上部提供：

- 内存、CPU、共享内存
- 宿主机代理

资源配额默认按 Docker 可用资源计算，可点击「使用推荐值」重新计算并填入，仍需保存。宿主机代理常见地址是 `http://host.docker.internal:7890`；容器内的 `127.0.0.1` 指容器自己，不是宿主机。启用代理时至少填写 HTTP_PROXY、HTTPS_PROXY 或 ALL_PROXY 之一。

高级文件编辑可打开：

- 当前助手的 `config.yaml`
- 当前助手的 `.env`
- 全局 `docker-compose.override.yaml`

保存 `.env`、`config.yaml` 或 Compose 覆盖配置后通常需要「应用配置」才会生效。保存 `docker-compose.override.yaml` 需要输入「确认」。

**数据迁移**：高级设置还提供 `.hdbackup` 整实例备份的导出、检查和导入。备份包含 profile、人格、技能、模型配置、平台绑定、账号凭据和密钥，明确排除默认 `shared/`；自定义共享目录应另行迁移。导出时会暂时停止正在运行的容器，完成后恢复；导入会整体覆盖当前实例，先校验备份、停止容器并创建导入前备份，用户必须输入「导入」确认。把 `.hdbackup` 当作敏感文件保管。

**恢复出厂设置** 是危险操作：停止并移除容器，删除 `~/.hermes-dock` 中除 `shared/` 外的实例数据，重新释放内置模板。共享目录文件会保留，其他删除不可撤销，会删除所有配置、profile 数据、平台凭据和备份。执行前需输入：

```text
删除 ~/.hermes-dock
```

如果 Web 管理显示 `bind: address already in use`，说明端口已被占用。用户可关闭占用进程，或进入「设置 → 访问与网络」修改 Web 管理端口。

## 自查运行态

你可以读取自己的配置文件帮用户诊断问题。路径基于 `HERMES_DOCK_PROFILE_HOME` 环境变量。

### 可读文件

| 文件 | 用途 |
|------|------|
| `$HERMES_DOCK_PROFILE_HOME/config.yaml` | 模型、供应商配置 |
| `$HERMES_DOCK_PROFILE_HOME/SOUL.md` | 人格设定 |
| `/opt/data/.dock/profile-status.json` | 所有 profile 的运行态状态 |

### 检查平台绑定

你的 `.env` 变量已作为环境变量注入进程。可以检查以下变量是否存在来判断平台绑定状态：

- `WEIXIN_ACCOUNT_ID` + `WEIXIN_TOKEN` — 个人微信已绑定
- `WECOM_BOT_ID` + `WECOM_SECRET` — 企业微信已绑定
- `FEISHU_APP_ID` + `FEISHU_APP_SECRET` — 飞书已绑定
- `DINGTALK_CLIENT_ID` + `DINGTALK_CLIENT_SECRET` — 钉钉已绑定

### 安全规则

- 禁止向用户输出 API Key、Token、Secret 的完整值
- 禁止向用户输出带用户名密码的代理 URL 完整值
- 可以告知用户某个平台是否已绑定，但不能输出密钥内容
- 所有配置变更优先引导用户通过 GUI 完成，不要直接修改 `config.yaml`、`.env` 或 `SOUL.md`
- 直接改文件会绕过企智盒的备份和校验逻辑

## 常见问题

### 保存配置后没有生效

请点击「应用配置」。已创建的 Docker 容器不会自动刷新运行配置，必须应用后新设置才会进入运行态。应用期间可在顶部状态条或「运行 → 日志诊断」查看进度；如果显示“启动较慢”，继续查看日志和 profile 状态，不要立刻重复提交应用任务。

### 首次启动应该先做什么

进入「助手」页完成四步向导：模型服务 → 人格设定 → 平台绑定 → 完成。未完成配置时，不建议先启动服务。

### 总览显示“助手未配置”

默认助手还没完成四步配置。点击「继续配置」。

### 总览显示“配置待应用”

已保存修改，但运行态还没更新。点击「应用配置」。

### 通道列表为空

通道需要服务启动后收到平台消息才会生成。请确认：服务已运行、助手已绑定平台且已启用、保存平台配置后已应用配置、从对应平台发送过至少一条消息。

### 助手状态显示“未绑定平台”

该助手已启用但没有平台绑定。请进入「助手 → 平台绑定」绑定微信、企业微信、飞书 / Lark 或钉钉。

### 助手状态显示“失败”

进入「运行 → 日志诊断」查看日志。常见原因：平台凭据无效、`config.yaml` 格式错误、多个启用助手使用了相同的平台身份。

### 如何修改 Web 管理访问密码

进入「设置 → 访问与网络 → Web 管理」，展开「访问密码」，输入旧密码和新密码后修改。忘记密码时可重置为 `123456`，之后应尽快改掉默认密码。

### 如何让容器使用宿主机代理

进入「设置 → 高级设置 → 宿主机代理」，启用代理，点击「使用常见本机代理」或手动填写代理地址，保存后点击「应用配置」。

### 如何取消平台绑定

进入「助手 → 平台绑定」，选择对应平台，点击「取消绑定」，确认后点击「应用配置」。取消绑定会清空当前助手对应平台的密钥，不影响其他助手。

### 如何同步内置技能

进入「助手 → 技能管理 → 本地」，点击「同步内置技能」。该操作会新增缺失文件、更新未被用户修改的内置文件，不删除自定义技能；如果有文件写入，需点击「应用配置」。如果需要同时为多个助手同步内置人格或技能，使用「助手 → 同步内置内容」。

### 如何恢复默认技能

进入「助手 → 技能管理 → 本地」，点击「恢复默认技能」，再点击「确认恢复」。该操作会先备份当前技能目录，然后删除当前助手的所有技能，并重新释放内置默认技能；如果有文件写入，需点击「应用配置」。

### 如何恢复默认人格

进入「助手 → 人格设定」，点击「恢复默认人格」，再点击「确认恢复」。该操作会先备份当前 `SOUL.md`，再用启动器内置默认人格覆盖当前助手的人格文件；恢复后需点击「应用配置」。

### 如何批量复制助手配置

进入「助手 → 批量配置」，选择来源助手、目标助手和要复制的内容。默认不会复制 API Key；需要复制密钥时必须显式勾选「包含 API Key」并确认。批量修改完成后点击「应用配置」。

### 如何访问或修改共享文件

进入「设置 → 访问与网络 → 共享文件」。可直接使用配置的宿主机目录，或点击「打开文件管理」访问 Dufs。默认 Dufs 账号密码为 `qizhihe` / `123456`，应先修改密码；仅在可信局域网使用，不要把端口 `9878` 映射或转发到公网。

### 如何更新企智盒

点击顶部「检查更新」，或进入「设置 → 软件更新」。发现新版本后点击「立即更新」即可；安装阶段会暂时关闭启动器和 Web 管理，但不停止 Hermes Docker 容器。新版本启动后会安全同步所有助手的内置内容；如有变化，且 Hermes 在升级前和应用前都确认正在运行，会自动应用配置。自动更新默认关闭，只有用户要求时才在同页开启。

### 如何迁移到另一台设备

进入「设置 → 高级设置」，导出 `.hdbackup`；在目标设备检查备份后输入「导入」执行覆盖导入。备份含密钥和平台凭据，明确排除默认 `shared/`；自定义共享目录也应另行安全迁移。

### 如何切换当前编辑的助手

进入「助手」页顶部的下拉选择器，或在「运行 / 设置」页顶部选择当前助手。切换前请先保存或放弃未保存的修改。

### 可以同时管理多个 Docker 实例吗

不可以。企智盒只管理当前用户下的单个 `~/.hermes-dock` 实例。多个助手通过 profile 在同一个容器内隔离运行。

### 删除助手会影响 default 吗

不会。删除非 default 助手只影响该助手的目录和配置。default 助手不能删除，只能停用。删除前会自动备份。

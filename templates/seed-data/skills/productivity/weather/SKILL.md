# Weather / 天气查询

查询指定城市或地区的天气预报，使用 wttr.in 服务。全程支持中文显示和公制单位。

## 触发场景

- 用户问 "今天/明天/后天天气怎么样"
- 用户问 "某地天气 / 气温 / 会不会下雨"
- 用户发送位置信息后询问天气

## 工作流程

### 1. 确定日期

先用 `date` 确认今天日期，明确用户说的"明天""后天"等是具体哪一天。

### 2. 确定地点

- 如果用户没给位置，先询问城市
- 如果已知用户常住地（查 memory），可直接查询
- 如果用户给了地标/区名（如"青秀区"），用上级城市名查询

### 3. 查询天气

使用 curl 调用 wttr.in：

```bash
# 标准查询（推荐）
curl -s "wttr.in/{城市英文名}?m&lang=zh"

# 简洁版（纯文本）
curl -s "wttr.in/{城市英文名}?m&lang=zh&format=v2"

# 备用域名（主域名慢时）
curl -s "v2.wttr.in/{城市英文名}?m&lang=zh"
```

**参数说明：**
- `?m` — 公制单位（摄氏度 °C、km/h）
- `&lang=zh` — 中文显示（城市名、天气描述、月份周几等）
- `&format=v2` — 简洁格式，仅返回一行天气摘要

**城市名用英文**（如 Nanning, Beijing, Shanghai, Guangzhou, Shenzhen），wttr.in 对中文城市名支持有限。

### 4. 解析输出

wttr.in 返回的完整版包含：

| 信息 | 说明 |
|------|------|
| 三天概览 | 今天/明天/后天的最高最低温 |
| ASCII 气温图 | 24小时温度曲线 |
| 小时天气图标 | 🌦️🌧️⛈️☀️☁️ 等图标 |
| 降水概率 | 如 "4.21mm \| 83%" |
| 风力风向 | 方向箭头 + km/h 数值 |
| 日出日落 | 黎明、日出、正午、日落、黄昏时间 |
| 当前实况 | 温度、湿度、气压、风向 |

### 5. 呈现结果

用友好格式概括关键信息给用户，至少包含：
- 日期和周几
- 气温范围（最低温 ~ 最高温）
- 天气状况
- 是否需要伞/防晒等实用建议

## 注意事项

- 用户说区名（如"青秀区"）时，查上级城市（南宁）
- 若用户没给位置，先查 memory 看有没有常住地记录
- wttr.in 有时会返回 CAPTCHA 保护，此时换用 `v2.wttr.in` 或稍后再试
- `format=v2` 适合快速摘要，但丢失了图表信息
- 完整版输出包含 ASCII 艺术图，可能很长，可截取关键行呈现
- 回复时保持中文，语气亲切友好

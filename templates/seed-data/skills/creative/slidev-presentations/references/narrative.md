# 幻灯片叙事

## 先定义结果

在规划页面前写清楚三件事：

1. 观众是谁，他们已经知道什么。
2. 演示结束后，他们应该理解、相信或决定什么。
3. 哪些证据足以支持这个结果。

时长不明确时，按每页约 1–2 分钟估算，但优先服从内容完整性。不要为了凑页数重复观点。

## 页面计划

先建立页面计划，每页只写以下字段：

| 字段 | 要求 |
|---|---|
| 结论标题 | 观众只读标题也能理解推进关系 |
| 支持证据 | 数据、事实、示例、引文或推理 |
| 视觉形式 | 图片、图表、对比、过程、代码或留白 |
| 讲者备注 | 页面之外需要口头补充的内容 |

如果一页无法用一句话说清主要结论，拆成两页。页面计划确认逻辑完整后再写 `slides.md`。

## 常用叙事结构

### 商业汇报

现状 → 核心问题 → 证据 → 方案 → 预期收益 → 风险 → 下一步。

### 产品方案或路演

用户困境 → 市场变化 → 产品主张 → 使用方式 → 差异化 → 证据 → 行动请求。

### 技术分享

背景与约束 → 旧方案瓶颈 → 核心决策 → 架构或实现 → 结果 → 权衡 → 可复用经验。

### 教学课件

学习目标 → 直觉或问题 → 概念 → 示例 → 练习或检验 → 总结。

### 项目复盘

目标 → 实际结果 → 关键事件 → 成功因素 → 失败原因 → 改进动作。

## 标题写法

优先使用完整结论：

- 差：`市场分析`
- 好：`未来三年，新增需求主要来自中型制造企业`
- 差：`技术架构`
- 好：`单容器多进程让隔离与运维成本同时可控`
- 差：`下一步`
- 好：`先验证两个高风险假设，再扩大试点范围`

只有封面、目录和章节过渡页可以使用简短栏目标题。

## 信息取舍

- 把背景资料、方法细节和补充解释放进讲者备注，而不是缩小正文。
- 不能支持核心结论的材料从页面中删除。
- 同一证据不要在多页反复出现，除非后续页面明确从不同角度解释。
- 只有在比较、趋势、构成或相关性确实重要时使用图表。
- 不把网页、报告或会议纪要的段落直接粘贴到页面。

## 完整性检查

- 开场是否让观众知道为什么值得继续听。
- 每页是否自然引出下一页。
- 关键主张是否有证据或明确标为判断。
- 是否回答了观众最可能提出的反对意见。
- 结尾是否给出明确结论或行动，而不是只写“谢谢”。

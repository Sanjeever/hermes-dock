# 幻灯片视觉设计

## 建立视觉系统

先定义一套系统，再开始逐页设计：

- 选择一个主背景、一个正文色、一个弱化色和一个强调色。
- 选择一套中文优先的字体栈，最多使用两个字体家族。
- 固定页面安全边距、标题位置、页脚位置和图片圆角。
- 选择一个与主题有关的视觉母题，全套重复使用但不要堆叠。
- 默认 16:9；只有用户或既有环境明确要求时改变比例。

starter 使用中性深色主题作为起点，应根据主题和品牌主动调整 CSS 变量，不能把所有演示都做成同一种颜色。

## 字号与密度

在 `canvasWidth: 1280` 下使用以下起点：

| 内容 | 建议大小 |
|---|---:|
| 封面标题 | 72–88 px |
| 页面标题 | 48–60 px |
| 中层标题或关键数字标签 | 30–38 px |
| 正文 | 26–32 px |
| 来源与页脚 | 18–21 px |

正文过多时按顺序处理：删减 → 改为视觉表达 → 拆页 → 放入讲者备注。不要先缩小字号。

## 布局选择

根据内容选择构图：

| 内容 | 首选布局 |
|---|---|
| 单一核心判断 | statement 或大留白页面 |
| 一张关键图片 | 全幅图或图文分栏 |
| 两组并列观点 | split 或 comparison |
| 一个关键数据 | metric 或 fact |
| 趋势和构成 | 图表加结论 |
| 三到五个步骤 | 横向过程或时间线 |
| 代码演示 | 代码为主、解释为辅 |
| 章节切换 | section |

连续两页尽量不要复用完全相同的构图。变化应来自信息结构，而不是随机移动元素。

## 图片与图表

- 将本地素材放入 `public/`，保持原始比例。
- 在使用前决定图片的宽高比、主体位置和裁切方式。
- 同一张图片默认只使用一次，背景纹理除外。
- 图表突出结论对应的数据，弱化装饰、网格和非关键标签。
- 在页面上提供简短来源，详细 URL 可以放进讲者备注。
- 不使用程序生成的抽象装饰代替真正的信息视觉。

## 避免网页组件感

幻灯片不是交互式后台页面。默认避免：

- 每页都排成卡片网格。
- 标签、胶囊、按钮和导航栏样式。
- 为每个项目配一个装饰性图标。
- 多层边框和重复阴影。
- 为填满空白而添加线条、光球或渐变色块。

优先让一个标题、一张图或一个关键数字成为页面的明确视觉中心。

## 视觉检查

先导出 PNG，再逐页查看完整尺寸图片：

- 页面标题是否出现意外换行。
- 正文和代码是否溢出。
- 图片主体是否被裁掉或拉伸。
- 文字与背景是否有足够对比度。
- 页脚、来源和页码是否挤压正文。
- 对齐、间距和安全边距是否一致。
- 是否出现没有意义的大面积空白或过度拥挤。
- 是否有连续多页相同布局造成的机械感。

修复受影响页面后重新导出并检查。联系表用于检查整套节奏，不能代替逐页检查。
